# TEST STRATEGY

## 1. Objective

The objective of this project is to validate the core functionality of the SauceDemo web application, ensuring a seamless user experience across critical e-commerce flows.

---

## 2. Scope

### In Scope

- Login functionality
- Product listing
- Add to cart
- Checkout process

### Out of Scope

- Performance testing (future enhancement)
- Security testing

---

## 3. Test Types

- Manual Testing
- Automation Testing (UI)

---

## 4. Test Approach

### Manual Testing

- Design test scenarios and test cases
- Cover positive, negative, and edge cases

### Automation Testing

- Focus on critical and repeatable flows:
  - Login
  - Add to cart
  - Checkout

---

## 5. Risk Analysis

| Feature  | Risk Level | Reason                   |
| -------- | ---------- | ------------------------ |
| Login    | High       | Entry point of system    |
| Checkout | Critical   | Revenue-impacting flow   |
| Cart     | Medium     | State-dependent behavior |

---

## 6. Test Environment

- Browser: Chrome
- OS: Windows
- URL: https://www.saucedemo.com/

---

## 7. Entry Criteria

- Application is accessible
- Test data is available

---

## 8. Exit Criteria

- All critical test cases executed
- No high/critical defects remaining
